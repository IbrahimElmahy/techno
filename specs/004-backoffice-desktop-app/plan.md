# Implementation Plan: Back-Office Desktop Application

**Branch**: `004-backoffice-desktop-app` | **Date**: 2026-06-28 | **Spec**: [spec.md](file:///d:/techno/specs/004-backoffice-desktop-app/spec.md)
**Input**: Feature specification from `/specs/004-backoffice-desktop-app/spec.md`

## Summary
The Back-Office Desktop Application is an installed Windows application built with Electron, React, TypeScript, and Ant Design. It functions as a thin client consuming the backend OpenAPI contract. In accordance with the project constitution, it contains no business logic, processes currency in EGP only, localizes fully to Arabic RTL, uses the Cairo font, and hides UI segments based on user role authorization while relying on server-side RBAC validation.

## Technical Context

- **Language/Version**: TypeScript 5.x, JavaScript (ES6+), React 18+, Node.js 18+
- **Primary Dependencies**: Electron, Ant Design (antd), React, Axios, openapi-typescript, electron-builder
- **Storage**: In-memory JWT storage, local config.json (external configuration)
- **Testing**: Vitest for component unit tests, Playwright for Electron application integration and UI tests
- **Target Platform**: Windows Desktop (installed x64 app)
- **Project Type**: desktop-app
- **Performance Goals**: Application launches in < 3 seconds; UI transitions are smooth and performant (< 200ms)
- **Constraints**: Default Arabic RTL layout, Cairo font, Light theme (primary green #6AB42D, accent orange #F5A11D), absolute reliance on server-side business logic, zero local data caching (except transient lookups)
- **Scale/Scope**: ~15 back-office pages consuming ~30 endpoints

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

| Principle | Status | Direct Implications on Design & Architecture |
|---|---|---|
| **I. Greenfield Only** | Pass | Built entirely from scratch in `frontend/`. No legacy code ported. |
| **II. Single Source of Truth** | Pass | Statically generated TypeScript typings from FastAPI's OpenAPI JSON to prevent drift. |
| **III. Offline-First Mobile** | Pass | N/A (Applies to Sales Rep mobile app only; this desktop app is online-only). |
| **IV. Reversibility** | Pass | Every return, deactivation, or manual ledger reversal requires explicit confirmation dialog. |
| **V. Multi-Branch & Multi-Warehouse** | Pass | UI lists items scoped by logged-in user branch (except admin). Forms select target warehouse. |
| **VI. Treasury & Customer Accounts** | Pass | Balances shown in UI are derived dynamically from ledger postings; no client-side balance addition. |
| **VII. Role-Based Access Control** | Pass | App shell hides forbidden menu items. Server-side endpoints strictly authorize every request. |
| **VIII. Arabic RTL, EGP Only** | Pass | Ant Design `ConfigProvider` sets RTL layout. Font family is Cairo. EGP is the only currency symbol. |
| **IX. Reporting Is First-Class** | Pass | Dedicated reports dashboard with robust filters (customer, territory, supplier, date, rep). |
| **X. Test-First for Critical Logic** | Pass | Core frontend routing, auth tokens, update checking, and response parsing will have written tests. |
| **XI. No Negative Stock** | Pass | Backend returns detailed errors when stock goes negative; client surfaces these as error toasts. |

## Proposed Changes

### [Frontend Core]
Implements the base configuration, Electron launcher, API layers, and general layouts.

#### [NEW] [package.json](file:///d:/techno/frontend/package.json)
Sets up Node dependencies for React 18, Ant Design, Electron 30, Axios, `openapi-typescript`, and compilation scripts.

#### [NEW] [vite.config.ts](file:///d:/techno/frontend/vite.config.ts)
Vite bundle settings using `vite-plugin-electron` to load electron main/preload scripts.

#### [NEW] [config.json](file:///d:/techno/frontend/config.json)
Configures the API URL. IT staff can modify this file directly in production.

#### [NEW] [electron/main.ts](file:///d:/techno/frontend/electron/main.ts)
Main electron script. Boots the window, reads `config.json` at runtime, checks for update version from server settings, and exposes hooks.

#### [NEW] [electron/preload.ts](file:///d:/techno/frontend/electron/preload.ts)
Exposes config properties and update checks to the renderer via `contextBridge`.

#### [NEW] [src/index.css](file:///d:/techno/frontend/src/index.css)
Imports Cairo font from Google Fonts and applies global reset classes.

#### [NEW] [src/App.tsx](file:///d:/techno/frontend/src/App.tsx)
Sets up Ant Design `ConfigProvider` with RTL direction, Cairo font-family, and primary green theme configuration.

#### [NEW] [src/api/client.ts](file:///d:/techno/frontend/src/api/client.ts)
Axios instance utilizing dynamic server URL from `window.electronAPI.getConfig()`. Appends session JWT to every request, intercepts 401/403 to trigger auto-logout, and handles error responses (negative stock warnings) via unified toast notifications.

#### [NEW] [src/components/AppLayout.tsx](file:///d:/techno/frontend/src/components/AppLayout.tsx)
Right-side collapsible navigation panel (RTL) displaying authorized menus based on role, with top bar (user profile, role details, and logout button).

Dashboard displaying daily, managerial, revenue, and purchase reports, filtered by date, rep, customer, and supplier.

## Verification Plan

### Automated Tests
- Component unit tests running in **Vitest**:
  - Verification of `ConfigProvider` applying RTL, Cairo font-family, and green primary color.
  - Mock API tests confirming Axios interceptors handle 401/403 (clears session and redirects to Login) and Axios appends JWT token.
- Desktop UI end-to-end integration tests using **Playwright**:
  - Emulates app launch with a mock `config.json`.
  - Simulates authentication sequence using user credentials.
  - Validates role-based navigation element visibility (e.g. log in as Sales Manager and verify Users/Treasury menus are hidden).
  - Tests negative stock validation interceptor (returns 400 with negative stock error, verifies toast error is visible).

### Manual Verification
- Deploying backend locally and running frontend in hot-reload mode.
- Changing `config.json` IP address to test connectivity loss behavior (error banner display).
- Verifying right-to-left layout alignment across modals and drawers in Arabic.
- Packaging the Electron app using `npm run build` and installing it on a Windows 10/11 machine to confirm installation flow.
